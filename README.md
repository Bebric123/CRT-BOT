# crt-bot

Telegram-бот для **предсказаний «на рабочую неделю»** в **чате обсуждения канала**: пользователь **упоминает бота** в комментарии под постом, в котором есть заданный **хештег-триггер** (по умолчанию `#predict_week`). Ответ — текст (опционально через локальную LLM) и случайная картинка из `assets/images` (подпись к фото, не текст на изображении).

## Требования

- Python 3.10+
- Токен бота от [@BotFather](https://t.me/BotFather)

## Установка

```bash
python -m venv venv
venv\Scripts\activate
pip install -r requirements.txt
```

Скопируйте `.env.example` в `.env` и заполните переменные (см. ниже).

## Настройка

| Переменная | Описание |
|------------|----------|
| `BOT_TOKEN` | Обязательно. Токен бота. |
| `ADMIN_USER_IDS` | Через запятую: числовые Telegram user id админов. |
| `WHITELIST_CHAT_IDS` | Необязательно: начальный whitelist **супергруппы обсуждения** (через запятую). |
| `SQLITE_PATH` | Путь к SQLite (по умолчанию `data/bot.db`). |
| `LOG_REJECTIONS` | `1` — писать в лог причины отказа. |
| `SILENT_REJECT` | `1` — при отказе не слать текст пользователю, только лог. |
| `LOCAL_LLM_*` | Опционально: генерация текста через OpenAI-compatible или Ollama API. |

Полный список и комментарии — в `.env.example`.

### Важно: whitelist и чат комментариев

У **канала** и у **связанной группы обсуждения** (куда попадают комментарии к постам) в Telegram **разные `chat_id`**. Бот проверяет id **именно группы обсуждения**.

Надёжный способ:

1. Зайти в группу обсуждения под каналом.
2. От имени админа выполнить **`/add_whitelist`** — в базу попадёт правильный id.

Проверка: **`/chat_id`** в этом же чате и **`/list_whitelist`**.

## Запуск

Из корня репозитория:

```bash
python run_bot.py
```

## Как пользоваться

1. В настройках канала включено обсуждение и бот добавлен в группу обсуждения (с правами на чтение сообщений и отправку ответов).
2. В посте канала есть настроенный хештег (например `#predict_week`).
3. В комментарии под этим постом пользователь **упоминает бота** (`@username_бота` или выбор из подсказки).

В группах с **темами** комментарий должен быть в **той же теме**, что и пост, либо цепочка ответов / кэш зеркала поста должны позволить связать сообщение с постом (см. `bot/discussion_cache.py`).

Лимит: повторное предсказание с того же аккаунта чаще чем раз в **60 секунд** блокируется (настраивается в коде `handlers.py`).

## Команды админа

Доступны только пользователям из `ADMIN_USER_IDS`.

| Команда | Действие |
|---------|----------|
| `/help_admin` | Список команд. |
| `/add_whitelist` | Добавить **текущий** групповой чат в whitelist. |
| `/remove_whitelist` | Убрать текущий чат из whitelist. |
| `/list_whitelist` | Показать сохранённые `chat_id`. |
| `/chat_id` | Показать `chat.id` текущего чата (удобно для обсуждения канала). |
| `/set_hashtag #тег` | Задать триггер-хештег в тексте поста. |
| `/get_hashtag` | Показать текущий тег. |
| `/bot_on` / `/bot_off` | Включить / выключить ответы бота. |
| `/status` | Краткий статус (enabled, hashtag, размер whitelist). |

Команды `/start` доступны всем.

## Картинки

Файлы **`.png` / `.jpg`** кладите в `assets/images/`. Случайный файл отправляется как фото; длинный текст при необходимости дополняется следующими сообщениями. Если папка пуста — бот отвечает только текстом.

## Локальная LLM

При `LOCAL_LLM_ENABLED=1` текст предсказания может генерироваться через HTTP:

- **`openai`** — `POST {LOCAL_LLM_BASE_URL}/chat/completions` (LM Studio, vLLM, Ollama в режиме OpenAI).
- **`ollama`** — `POST {LOCAL_LLM_BASE_URL}/api/chat` (базовый URL обычно `http://127.0.0.1:11434`).

При ошибке или пустом ответе используется встроенный шаблонный текст.

## Структура проекта

```
crt-bot/
  run_bot.py          # точка входа
  bot/
    main.py           # polling, регистрация хендлеров
    handlers.py       # упоминания, ответы, админ-команды
    discussion_cache.py  # кэш постов канала по треду (хештег без обязательного reply)
    storage.py        # SQLite: whitelist, настройки, rate limit
    config.py         # загрузка .env
    prediction.py     # шаблоны, картинки, нарезка текста
    local_llm.py      # клиент LLM
  assets/images/      # картинки для ответов
  data/               # SQLite (по умолчанию)
```

## Отладка

- **`SILENT_REJECT=0`** — при отказе пользователь увидит причину (whitelist, выключенный бот, нет хештега в контексте поста и т.д.).
- В логах при `LOG_REJECTIONS=1` пишутся строки `reject chat_id=... reason=...`.
- У бота в группе должен быть доступ к сообщениям; при **режиме приватности** бот в группах видит не все сообщения — для срабатывания логики обычно достаточно **явного упоминания** бота.
